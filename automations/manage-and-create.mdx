title: "Manage and Create Automations"
description: "Use the Automations page to find drafts, create new work, and move into the builder."
---

The `Automations` page is the entry point for draft automation management.

## What you can do from the Automations page

- search for an existing draft;
- sort cards by age;
- open an automation for editing;
- duplicate a draft;
- delete a draft;
- create a brand-new automation.

## Find the right automation

1. Open `Automations`.
2. Use `Search...` to narrow the card list.
3. Use `Sort By` to switch between `Newest first` and `Oldest first`.
4. Open the matching card.

If the workspace has no automations yet, Magical shows `No automations`.

## Create a new draft

1. Click `Create`.
2. Magical creates a new draft automation.
3. You are routed directly into the builder at `/automations/{id}`.

There is no separate create modal in the current flow.

## Name the draft and orient yourself

As soon as the builder opens:

1. rename the draft in `Automation Name`;
2. use `Build` to work on the canvas;
3. use `Test` to validate saved behavior over time;
4. use `Settings` for schedule, notifications, credentials, import/export, and other published behavior.

## Card actions

Use the card menu when you need a draft-level action:

- `Edit` opens the draft.
- `Duplicate` makes a copy.
- `Delete` removes the draft after confirmation.

Use deletion carefully. It affects the draft record itself, not just a version inside the builder.

## Next steps

After creation:

1. add the starting node and supporting nodes;
2. configure inputs, outputs, tools, and instructions;
3. validate the draft with `Try it` or automation tests;
4. publish only when the draft is ready for normal runs.

The build workflow continues in [Build on the Canvas](/automations/build-on-the-canvas).
