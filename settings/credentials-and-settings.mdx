Automation settings are the hub for the workflow behavior that sits around the canvas.

## Automation settings surfaces

From `Settings`, you may see panels such as:

- `Run automatically`
- `Notifications`
- `Credentials`
- `Import or export agents`
- `Workflow billing`
- `Unpublish`
- `Delete automation`
- `Published ID`

Some panels only appear after an automation is published.

## Create credentials in workspace settings

Credentials are workspace-level resources. Create them before trying to select them inside an automation.

1. Open `Workspace settings`.
2. Go to `Authentication`.
3. Click `Add`.
4. Choose the authentication type.
5. Complete the required fields or external connection flow.
6. Confirm the credential appears in the authentication table.

Common authentication types include:

- `Cookie`
- `1Password`
- `Phone Auth Code`
- `Email Auth Code`
- `API connection`
- `Desktop connection`
- `Native App connection`
- `Remote Desktop Gateway`
- `Proxy server`

## Use credentials inside an automation

Credentials appear in two places:

- inside an API node's `Authentication` field;
- in the automation `Settings` > `Credentials` panel.

For an API node:

1. select the node;
2. choose the integration;
3. open `Authentication`;
4. select the saved credential;
5. save the automation.

If no credential exists, use the link back to workspace settings and create it there first.

## Review automation credentials

The `Credentials` panel in automation settings lets you review or remove credentials attached to the automation.

Use it carefully:

- removing a credential can affect nodes that rely on it;
- this panel is not the primary place to create new credentials;
- available credentials change with organization context.

## Schedule a published automation

Use `Run automatically` for interval scheduling.

1. open the automation;
2. go to `Settings`;
3. turn scheduling on;
4. choose the interval number;
5. choose `minutes`, `hours`, or `days`;
6. save the change.

Common blockers:

- the latest published version has no trigger;
- schedule data failed to load;
- the workflow uses a calendar-style schedule that cannot be edited in the portal.

## Configure notifications

Notifications are for published automation activity.

### Email

1. open the `Email` section;
2. add recipients in `Type to add email addresses`;
3. remove recipients that should not receive updates.

### Slack

1. open the `Slack` section;
2. connect Slack if it is not already connected;
3. choose the destination channel;
4. disconnect only when the workspace should stop sending Slack notifications.

If email notifications are enabled with no recipients, Magical shows a validation warning.

## Pre-publish settings checklist

Before publishing or republishing:

1. confirm the right credentials are attached;
2. confirm notifications go to the right people and channels;
3. confirm scheduling is either intentionally on or intentionally off;
4. review any billing fields if the workflow requires them.

## Important boundaries

- Schedule and notification settings require a published automation.
- Credential creation belongs in workspace settings.
- Calendar scheduling currently has limited edit support in the portal.
- Settings changes can alter how the published automation behaves, even when the canvas logic is unchanged.
