---
title: "Practice: Triage and Review a Run"
description: "Practice run review by deciding whether a run needs feedback, approval, rerun, or no action."
---

Use this exercise after learners understand the run log, run detail page, live supervision, reruns, and checkpoints.

## Practice goal

Triage a run as if you are the builder responsible for deciding what happens next.

## Scenario

A teammate asks you to review a customer-support automation run.

The run may have:

- completed normally;
- paused for human action;
- failed;
- moved into a needs-attention state.

Your job is to inspect the run, explain what happened, and choose the next action.

## Exercise

1. Open `Home` and find the relevant run, or open the run directly.
2. Decide whether it is in `Needs attention`.
3. Open the run detail page.
4. Review the status and activity feed before taking action.
5. Identify the first action or step that best explains the current state.
6. Decide whether the right response is feedback, approval, retry, rerun, stop, or no action.
7. If the workflow includes checkpoints, review the checkpoint evidence.
8. Write a short handoff note that explains what happened and what should happen next.

## Completion criteria

The learner should be able to explain:

- the current run state in plain language;
- why the run does or does not need attention;
- which run step matters most;
- what the next action should be;
- whether checkpoint evidence changes the decision;
- what they would tell a teammate next.

## Coach checklist

A strong submission shows:

- a plain-language explanation of status;
- one cited activity item or run step;
- a clear next action;
- no unnecessary destructive action;
- specific feedback when feedback is needed;
- checkpoint review when checkpoints are part of the workflow;
- a concise teammate handoff note.

## Important boundaries

- This is a reasoning exercise, not a separate product surface.
- Internal diagnostics or verification pages are not part of the normal run-triage path.
- If a learner cannot access a real run, they can still complete the exercise using the topic guides.
