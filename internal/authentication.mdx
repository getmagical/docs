---
title: Authentication
description: Get access tokens for the Magical REST and GraphQL APIs.
---

All REST and GraphQL calls require a Bearer access token.

```http
Authorization: Bearer <accessToken>
```

## Get a token (staging or production)

<Steps>
  <Step title="Request a login code">
    Magical emails a one-time code to the address you submit.

    ```bash
    curl -X POST https://api.getmagical.com/auth/request-code \
      -H "Content-Type: application/json" \
      -d '{"email": "user@example.com"}'
    ```
  </Step>
  <Step title="Exchange the code for tokens">
    ```bash
    curl -X POST https://api.getmagical.com/auth/login \
      -H "Content-Type: application/json" \
      -d '{"email": "user@example.com", "code": "received-code"}'
    ```

    The response contains `accessToken` and `refreshToken`. Use the access token in the `Authorization` header for subsequent requests.
  </Step>
</Steps>

## Get a token (local dev)

Locally, any code is accepted:

```bash
curl -X POST http://main.localhost:1355/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "code": "any-code"}'
```

## Organization impersonation

Users and tooling that span multiple organizations (for example, internal ops tools) can target a specific organization by adding the `X-Organization-Id` header:

```bash
curl https://api.getmagical.com/automation \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "X-Organization-Id: org_123456789"
```

<Warning>
  `X-Organization-Id` is enforced server-side. Only callers with permission to act across organizations can use it; regular users are scoped to the org on their token.
</Warning>

## Refreshing tokens

Use the `refreshToken` returned at login to get a new access token before it expires. The refresh exchange endpoint is exposed under `/auth` in `service-main`.

## Webhook authentication

Inbound webhooks that trigger automations use per-automation **webhook secrets** and either shared-secret Bearer tokens or HMAC signatures, not user access tokens. See [Webhook triggers](/internal/webhook-triggers).
