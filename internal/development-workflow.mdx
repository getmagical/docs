## Run everything in watch mode

```bash
turbo watch dev
```

Or, with Doppler env vars loaded:

```bash
turbo watch dev:load-env
```

<Tip>
  Prefer `:load-env` variants when available so env vars are loaded consistently.
</Tip>

## Run a single service

Each package can run standalone by filter:

```bash
turbo watch dev --filter=@magical/service-main
turbo watch dev --filter=@magical/app-portal
```

For hot reloading of agent-related modules in `remote-session`, run it in its own terminal:

```bash
turbo watch dev --filter=@magical/remote-session
```

## Override env vars locally

Drop a `.env` file into the package you're running to override Doppler values:

```bash
turbo watch dev:load-env --filter=@magical/package-name
```

## Storybook

```bash
cd packages/design-system && npm run storybook   # Port 6006
cd apps/portal && npm run storybook              # Portless URL
```

Portal Storybook is reachable at `http://storybook.localhost:1355` (primary) or `http://<worktree>.storybook.localhost:1355` (linked worktrees).

<Note>
  With watch mode, code changes auto-apply. No restart is needed. Check logs for errors.
</Note>

## Quality checks

```bash
npm run check                                   # Lint, format, types with auto-fix (parallel)
npm run check -- --filter=@magical/app-portal   # Scoped (turbo flags like --force ok)
npm run test:unit                               # Unit tests
npm run test:int                                # Integration tests (needs DB)
npm run test:e2e                                # End-to-end tests (needs DB)
npm run build                                   # Build all
```

## Troubleshooting a bad build

```bash
npm run clean && npm ci && npm run build   # Clear cache, rebuild
npm run package-lock:regenerate            # Fix dependency issues
```

For deeper troubleshooting of Doppler and Turborepo cache, see [Troubleshooting](/internal/troubleshooting).

## Default local URLs

| Surface | Primary | Linked worktree |
| --- | --- | --- |
| Portal | `http://portal.localhost:1355` | `http://<worktree>.portal.localhost:1355` |
| service-main | `http://main.localhost:1355` | `http://<worktree>.main.localhost:1355` |
| Worker | `http://worker.localhost:1355` | `http://<worktree>.worker.localhost:1355` |
| Portal Storybook | `http://storybook.localhost:1355` | `http://<worktree>.storybook.localhost:1355` |
| MCP | `http://mcp.localhost:1355` | `http://<worktree>.mcp.localhost:1355` |

Portal and portal Storybook can auto-open the Portless URL once the app route is ready.

## `inject-envs`

If you need `inject-envs` and don't have it on `PATH`, invoke it with `npx`:

```bash
npx inject-envs -- <command>
```
