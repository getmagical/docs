---
title: Agent configuration
description: Configure AI providers for agents via `.agents/settings.local.json`.
---

You can customize which AI providers run agents by creating a `.agents/settings.local.json` file. This file is gitignored and allows per-developer customization.

## Minimal example

Use each agent's recommended provider:

```json
{}
```

## Full example

```json
{
  "provider": {
    "default": "claude",
    "allowlist": ["claude", "codex", "gemini"]
  },
  "agents": {
    "customConfigs": {
      "code-reviewer": {
        "provider": "gemini",
        "model": "gemini-2.5-pro"
      },
      "planner": {
        "provider": "codex",
        "model": "gpt-5.2-codex",
        "reasoningEffort": "high"
      }
    }
  }
}
```

## Options

### `provider`

| Field | Description |
| --- | --- |
| `default` | Default provider to use when the requested provider is not available. Must be one of `claude`, `codex`, `gemini`. Must be in `allowlist` if that's set. **Required** when `provider` is specified. |
| `allowlist` | Array of providers you have access to. If set, only these providers can be used. If an agent's recommended provider is not in this list, `default` is used instead. If not set, all providers are allowed. |

### `agents.customConfigs`

Per-agent overrides:

| Field | Description |
| --- | --- |
| `provider` | Override the agent's recommended provider (must be in `allowlist` if set). |
| `model` | Specify the exact model to use. |
| Other | Additional provider-specific config (for example, `reasoningEffort` for Codex). |

Custom agent configs are merged with and override the agent's default configuration.

## Provider resolution precedence

The system resolves which provider to use in this order (highest to lowest):

1. **Custom agent provider** — `agents.customConfigs[agent].provider`, if specified.
2. **Default provider fallback** — `provider.default`, if the agent's recommended provider is not in `provider.allowlist`.
3. **Agent's recommended provider** — the default from the agent's frontmatter `recommended-provider`.

<Note>
  The settings file is validated with Zod. Invalid configurations produce clear error messages with examples.
</Note>
