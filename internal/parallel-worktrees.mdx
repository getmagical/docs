---
title: Parallel worktrees
description: Use git worktrees and Portless to develop multiple features side by side without port collisions.
---

Git worktrees let you work on multiple features in parallel from the same clone. The monorepo's dev setup automatically derives a unique Portless namespace per worktree so local service URLs don't collide.

## Create a worktree

<Steps>
  <Step title="Branch from develop into a sibling directory">
    ```bash
    git worktree add -b <new-branch-name> ../<new-directory-name> origin/develop
    ```

    Example:

    ```bash
    git worktree add -b feature-a ../wt-feature-a origin/develop
    ```
  </Step>
  <Step title="Open the worktree in Cursor">
    ```bash
    cursor ../wt-feature-a
    ```
  </Step>
  <Step title="Create a fresh DB branch scoped to this worktree">
    ```bash
    npm run new-db
    ```
  </Step>
</Steps>

## What the dev scripts do for you

`scripts/inject-dev-env.sh` automatically:

- Derives a stable namespace for linked worktrees.
- Exposes local services through Portless URLs (default proxy port `1355`):
  - Primary workspace: `http://portal.localhost:1355`, `http://main.localhost:1355`, `http://worker.localhost:1355`
  - Linked worktrees: `http://<worktree>.portal.localhost:1355`, `http://<worktree>.main.localhost:1355`, `http://<worktree>.worker.localhost:1355`
- Sets up unique Temporal queues so workflows from different worktrees don't collide.

<Tip>
  Pair worktrees with per-worktree Neon branches (`npm run new-db`) so your local database state is isolated too.
</Tip>
